拡張保証

Archetype Guarantee

agent selection、operator guidance、conflict、assessment、replay explanation のいずれかに現れる actor / role archetype が最低1つある。

World Guarantee

world rule、pressure、layer、institution、ecology、machine system、boundary condition のいずれかが最低1つある。

Operator-entry phrase は internal design input として扱えるが、public code に copyrighted catchphrase、 character name、quoted line を hard-code しない。

Minimum World Richness Guarantee

  • Place and environment
  • Rules of possibility
  • Social fabric
  • Resources and power
  • History and memory
  • Daily life signal
  • Change pressure

7項目を満たせない addition は、parking-lot または watch item としてのみ扱う。

Fractal Decision / FDE

default decision shape は three AI branches plus one meta-user oversight layer。 Proposal branch、Review branch、Execution branch、Fourth-power oversight に分ける。

Entry
decision を trigger したもの。
Packet
smallest reviewable unit。
Evidence
local files、tests、sources、explicit assumptions。
Decision
adopt、revise、park、reroute、reject、escalate。
Closure
何が変わり、何が残り、誰が reopen できるか。

Scale-Simplification Simulation Benchmark

外部公開post由来の採用候補として、scale を広げると simulation が単純化する場合がある、 という benchmark を追加する。引用ではなく抽象化した評価軸として扱う。

Social Noise
人間社会の個別具体は noise が多く、local simulation が難しい場合がある。
Physical Scale
物理法則が支配する大きな scale では、rule set が少なくなり model 化しやすい場合がある。
Cognition Boundary
難しさは world 側だけでなく、人間の認知・理解・観測粒度にも依存する。
Layer Comparison
micro social layer と macro physical layer を比較できるようにする。

Worldbuilding Extraction Pipeline

別projectから採用するのは内容ではなく pipeline のみ。private path、固有記事、固有世界名、 本文断片は public docs に持ち込まない。

Section Split
source article を section に分解する。
Layer Classify
section を world layer に分類する。
Claim Extract
claim candidate を抽出する。
Claim Metadata
subject、predicate、claim kind、source、terms を付ける。
Vocabulary
terms を concept vocabulary として集約する。
Validation
wiki-style link / reference の欠落を検証する。
Orchestration Report
生成手順、抽出数、検証結果を残す。

Non-goals / Gates

  • fictional characters の再現や protected lines の引用はしない。
  • internal notes、private paths、voice-chat transcripts は public docs に出さない。
  • Discord から GitHub へ auto-write しない。
  • P2P は design spike 先行。network implementation には入らない。
  • cloud commands、cloud resources、secrets、billing / quota changes はこの PR では扱わない。
  • follow-up implementation には separate approved work order が必要。

Source Categories

public-policy

external-practice
external product / community-management practice に grounded。short source link または review note を残す。

この project の local operating rule。real review cycle で試すまで adjustable とする。
creative-hypothesis
original motif synthesis / speculative design。industry practice として提示しない。

初回 PR Scope

初回 PR は docs-only。application code、data contracts、generated docs、tests は変更しない。 GitHub / Linear / Discord / cloud / external API の write actions は human review の後に分離する。
