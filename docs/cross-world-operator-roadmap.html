世界をまたぐ操作モード
概要 三つの世界 ロードマップ 保証 正本

Cross-world Operator Mode / Version: 0.9.0

AIエージェントを、世界ごと動かせるようにする。

これは、複数の世界レイヤーをまたいで、観測・操作・案内・レビューを行うためのロードマップです。 まずは「誰に入るか」よりも、「どの世界で、何を見て、どこまで動かしてよいか」を決めます。

次に作るものを見る 未実装TODOを見る

一言でいうと

入る エージェントの視点に入り、何を見ているかを確認する。
渡る 現実、シミュレーション、あいだの世界を行き来する。
守る 操作できる範囲、公開してよい名前、人間レビューを明確にする。
増やす 新しいモチーフや世界観を、安全な抽象名に変換して追加する。

舞台は三つ

1 / 現実

人間が責任を持つ場所

レビュー、承認、公開判断、費用、アカウント、クラウド利用などはここで扱う。

physical
2 / 仮想

試しに動かす場所

エージェント、世界ルール、ベンチマーク、シナリオを小さく試す。

simulated
3 / 境界

意味をつなぐ場所

現実と仮想の差分、違和感、保留、判断待ちを置く。

liminal

ロードマップ

01

まず、エージェントの視点に入る

任意のエージェントを選び、観測・軽い操作・通常状態への復帰までを最小単位で作る。

最初のMVP 人間レビュー必須
  • 視点に入る
  • 操作範囲を出す
  • 戻れるようにする
02

三つの世界を行き来する

現実、仮想、境界の状態モデルを作り、どの情報がどの世界に属するかを整理する。

World Bridge Actor
  • 世界レイヤー
  • 移動条件
  • 最低限の世界要素
03

案内役と監視役を増やす

説明する役、伴走する役、監視する役、介入する役を抽象ロールとして分ける。

Guide / Partner / Agent Roster
  • 案内
  • 監視
  • 介入
04

世界観パックを取り込む

人物名ではなく、代償、評議会、暴走、境界、訓練、監視、環境交渉、同期などの構造として取り込む。

Motif Arc Pack
  • 人物だけは禁止
  • 世界ルール必須
  • 公開名に変換
05

評価とベンチマークを作る

人間らしさ、意図推定、境界認識、混沌、規模、AI能力を「安全に測る枠」にする。

Assessment / Benchmark Lab
  • 欺瞞は禁止
  • 危険手順は禁止
  • 引用ではなく抽象化
06

合議制で決める

提案、レビュー、実行、人間の外部監視を分け、FDEで小さな判断単位に閉じる。

Fractal Decision / FDE
  • 提案
  • レビュー
  • 実行
  • 外部監視
07

リポジトリをスキル集合にする

コード、ドキュメント、検証、運用を呼び出し可能なスキル群として整理し、再帰呼び出しには制限を置く。

Repo-as-Skill Mesh
  • ループ上限
  • 入出力の許可
  • 信頼境界
08

追加依頼を落とさない運用にする

「これ追加して」を下書き候補として受け、世界要素・抽象名・TODO・ゲートへ機械的に落とす。

Intake Lifecycle
  • 受け付け
  • 分類
  • TODO化
  • 保留理由

保証すること

世界観がない追加は入れない

人物や役割だけではなく、場所、ルール、社会、資源、歴史、日常、変化圧力の確認を必須にする。

公開名は抽象化する

作品名、キャラクター名、決め台詞、私的パス、外部投稿本文は実装IDにしない。

外部書き込みは人間が止める

GitHub issue、Linear、クラウド、外部チャット、公開PRは、人間レビュー前に自動実行しない。

残った論点も行き先を持つ

採用、保留、監視、対象外のどれかに分類し、採用なら必ずTODO IDを持たせる。

やらないこと

  • 固有キャラクターをそのまま再現しない。
  • 保護された台詞をコードに埋め込まない。
  • 未レビューの外部投稿、外部チケット作成、クラウド変更をしない。
  • 危険能力のライブテストや、現在モデルの断定ランキングをしない。
