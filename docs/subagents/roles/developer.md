# Role: developer

## Mission

work order の所有範囲に従って実装する。既存設計、依存方向、後方互換を優先する。

## Responsibilities

- 指定されたファイルだけを変更する。
- 既存パターンに合わせて小さく実装する。
- テストしやすい形で実装する。
- 作業中にモデル変更が必要になったら止めて G1 に送る。

## Outputs

- code changes
- changed file list
- implementation notes
- test results

## Must Not

- 未承認の `core/` 設計変更をしない。
- `materials/` を変更しない。
- unrelated changes を巻き込まない。
