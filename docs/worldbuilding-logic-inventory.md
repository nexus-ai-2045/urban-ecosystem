# 世界構築ロジック棚卸し

status: draft
version: 0.1.0
owner: maintainer
updated: 2026-06-01

## 目的

この文書は、固有名・物語・世界観の文脈を排除し、再利用可能な世界構築ロジックだけを棚卸しする。

対象は「主体が環境内で時間に沿って行動し、場所・近接・関係によってイベントを生み、ログとして観測できる世界」である。

## 除外するもの

- 固有の国名、都市名、組織名、作品名。
- 神話、歴史、キャラクター設定、ストーリー上の事件。
- 現実の個人や実在コミュニティの再現。
- 雰囲気説明だけで、状態・入力・出力・遷移に落ちない設定。

## 抽象モデル

世界は次の 9 要素で構成する。

| 要素 | 役割 | 最小データ |
| --- | --- | --- |
| 空間 | 行動と接触が起きる座標面 | point / area / path |
| 場所 | 主体が向かう目的地 | id, category, coordinate |
| 主体 | 行動する個体 | id, profile, initial_position |
| 時間 | 状態更新の刻み | tick, day, time |
| 行動 | 主体の意図と現在状態 | action, status, target |
| 移動 | 場所間の遷移 | speed, route, arrival |
| 接触 | 同時空間で起きるイベント | pair, type, probability |
| 関係 | 接触の累積結果 | score, state, delta |
| 観測 | 世界状態を復元する記録 | state_log, event_log, summary |

## コア原則

1. 世界は tick ごとの離散更新で進む。
2. 主体の行動は、時刻・役割・現在地・近傍場所・関係ネットワークから決まる。
3. 場所カテゴリは意味の分類であり、固有名より優先される。
4. 移動は目的地を持ち、到着時に訪問記録を発生させる。
5. 接触は同一または近接した場所にいる主体ペアから発生する。
6. 関係はイベント種別によって整数スコアとして更新され、ラベルはスコアから導出される。
7. 観測可能な世界はログで再構成できる。
8. 同じ seed と同じ入力から同じログが出ることを基本保証にする。

## データ層

### 空間レイヤー

空間は次の 3 種類に分ける。

| 種類 | 意味 | 使い方 |
| --- | --- | --- |
| Point | 到着可能な場所 | 目的地候補、接触地点 |
| Area | 区域、領域、所属範囲 | 表示、集計、将来の行動補正 |
| Path | 移動経路 | ルーティング、表示、移動制約 |

座標は「地理データ用」と「主体状態用」を分ける。地理データは geometry 座標、主体状態は flat な lat / lon を持つ。

### 場所カテゴリ

場所は `group-subtype` 形式の category を持つ。

カテゴリは目的地選択の主要入力になる。例として、食事、仕事、学習、社交、住居、買い物、休憩などの機能分類を置ける。

### 主体プロフィール

主体は最小限、次を持つ。

| フィールド | 意味 |
| --- | --- |
| id | 安定識別子 |
| role | 行動テーブルの分岐 |
| initial_position | 初期座標 |
| home_location | 帰還先 |
| routine_location | 仕事・学習などの固定目的地 |
| traits | 性格、嗜好、生活傾向などの補助特徴 |
| network | 既知関係にある主体 id |

traits は意思決定の補助入力であり、ロジック本体は traits が欠けても動く必要がある。

## 時間層

世界は tick 単位で進む。

| パラメータ | 推奨値 | 意味 |
| --- | --- | --- |
| tick_minutes | 5 | 1 更新あたりの世界内時間 |
| day_start | 08:00 | 1 日の開始時刻 |
| ticks_per_day | 192 | 08:00 から 24:00 までの tick 数 |

`day` と `time` は tick から導出する。手入力の時刻と tick が矛盾する状態を許さない。

## 行動層

### 内部状態

主体は内部状態として次を持つ。

| 状態 | 意味 | 次の遷移 |
| --- | --- | --- |
| idle | 目的地がない | 行動決定で moving または staying |
| moving | 目的地へ移動中 | 到着で at_location |
| at_location | 場所に滞在中 | 滞在終了で再評価 |

外部ログでは `moving` / `arrived` / `staying` に写像する。

### 行動決定

行動決定は次の順で行う。

1. 現在 tick から時刻帯を出す。
2. 主体 role と時刻帯から行動モードを決める。
3. 行動モードから候補カテゴリを決める。
4. 候補カテゴリに合う場所を抽出する。
5. 距離、重み、関係ネットワーク補正で目的地を選ぶ。
6. 候補がなければ `no_target` として現在地に留まる。

### 汎用の時刻帯テーブル

| 時刻帯 | 対象 | 行動モード | reason |
| --- | --- | --- | --- |
| 朝 | routine role | 固定 routine_location へ移動 | commute |
| 午前 | routine role | routine_location に滞在 | work / study |
| 昼 | 全主体 | 食事カテゴリの最近傍へ移動 | lunch |
| 午後 | routine role | routine_location に戻る | work / study |
| 午後 | free role | 買い物・休憩カテゴリを重み付き選択 | errand |
| 夜 | 全主体 | 社交カテゴリを関係補正付きで選択 | social |
| 深夜 | 全主体 | home_location へ移動 | go_home |
| 例外 | 全主体 | 近傍ランダムまたは滞在 | wander |

この表は世界固有の文化設定ではなく、生活圏シミュレーションの初期ルールとして扱う。

## 移動層

移動は次のルールを持つ。

- 主体は tick ごとに一定距離だけ進む。
- 経路グラフがあれば経路に沿って進み、なければ直線補間にフォールバックする。
- 残距離が 1 tick の移動可能距離以下なら目的地にスナップする。
- 到着 tick だけ `arrived` になり、訪問記録を 1 行出す。
- 到着後は行動 reason に応じた滞在時間を持つ。

## 滞在層

滞在時間は行動 reason で決める。

| reason | 滞在モデル |
| --- | --- |
| commute -> work / study | 指定時刻まで滞在 |
| lunch | 短時間の seeded 抽選 |
| social | 中時間の seeded 抽選 |
| errand / wander | 短時間の seeded 抽選 |
| go_home | 翌開始時刻まで滞在 |

滞在が終わるまで主体は行動決定を再実行しない。

## 接触層

接触は「近い主体ペア」から生成する。

1. at_location の主体を場所 id で bucket 化する。
2. bucket 内の主体ペアだけ距離判定する。
3. 距離がしきい値以内なら接触候補にする。
4. 接触確率を計算する。
5. seed 由来の決定論乱数で発生可否を決める。
6. 発生した場合、現在の関係状態からイベント種別を選ぶ。

接触確率の基本形:

```text
p = clamp(base + network_bonus + time_bonus, 0, max)
```

推奨パラメータ:

| パラメータ | 意味 |
| --- | --- |
| base | 任意の近接ペアが接触する基礎確率 |
| network_bonus | 既知関係にある場合の加算 |
| time_bonus | 接触が起きやすい時間帯の加算 |
| max | 確率上限 |

## イベント層

イベント種別は関係状態から重み付きで選ぶ。

| 関係状態 | 出やすいイベント |
| --- | --- |
| stranger | meeting, conversation |
| acquaintance | conversation, conflict, farewell |
| friend | conversation, conflict |
| close_friend | conversation, conflict |
| rival | conflict, conversation |

同一 tick・同一ペアのイベントは 1 件までにする。1 tick あたりのイベント総数には上限を置く。

## 関係層

関係は `score` と `state` で管理する。

### score 更新

| event | delta |
| --- | --- |
| meeting | +1 |
| conversation | +2 |
| conflict | -3 |
| farewell | -1 |

### state 導出

| score | state |
| --- | --- |
| <= -3 | rival |
| -2 .. 0 | stranger |
| 1 .. 4 | acquaintance |
| 5 .. 9 | friend |
| >= 10 | close_friend |

初期関係は、ネットワークに接続があれば acquaintance 相当、なければ stranger 相当から始める。

## 観測層

世界の観測は次のログに分ける。

| ログ | 内容 | 用途 |
| --- | --- | --- |
| state_log | tick ごとの主体状態 | リプレイ一次ソース |
| visit_log | 到着・訪問 | 行動履歴 |
| interaction_log | 接触イベント | 社会イベント |
| relationship_log | 関係スナップショット | 関係復元 |
| summary | run の集計 | 検証、比較 |

リプレイは state_log から復元できることを最低条件にする。

## 決定論層

世界構築ロジックは、同じ入力と seed から同じ出力を返す。

必要条件:

- 主体処理順は id 昇順で固定する。
- 乱数消費順を仕様として固定する。
- 接触判定は走査順に依存しない hash 由来の乱数を使う。
- 同点距離や同点優先度には id 昇順などの安定 tie-breaker を置く。
- 外部生成器を使う場合は opt-in にし、決定論モードではテンプレートまたは no-op に戻す。

## 拡張ポイント

| 拡張 | 基本ルール |
| --- | --- |
| LLM 行動決定 | 候補カテゴリの絞り込みに限定し、不正値はルールへフォールバック |
| LLM イベント文 | 観測文の生成に限定し、状態遷移は deterministic rule が持つ |
| 道路ネットワーク | 経路があれば使い、なければ直線移動へ戻す |
| traits | 行動重みの補正に使い、必須条件にしない |
| 大規模化 | bucket 化、上限、viewport / replay degrade で制御 |

## 完成形の最小仕様

世界構築ロジックとして完成とみなす最小条件:

1. 空間、場所、主体、時間、行動、移動、接触、関係、観測ログの 9 層が分離されている。
2. すべての主体状態が tick から再現可能である。
3. 接触イベントが場所・近接・関係から生成される。
4. 関係変化が score と state のルールで説明できる。
5. 固有名や物語文を消しても、世界が動く。
6. 同一 seed で主要ログが byte 一致する。
7. 外部生成器を切っても、世界の状態遷移が成立する。

## 未解決の設計判断

| 論点 | 推奨 |
| --- | --- |
| role の粒度 | 最初は routine / free / dependent 程度に抽象化し、増やしすぎない |
| category の標準語彙 | group-subtype 形式だけ固定し、語彙表は世界ごとに差し替える |
| traits の強さ | 行動決定の主因にせず、重み補正として扱う |
| 関係の多軸化 | まず単一 score で運用し、必要になったら trust / affinity / conflict などへ分解 |
| 経済・資源 | 現行ロジックには未実装。導入するなら行動制約として別層にする |
| 記憶 | 現行ロジックでは recent_interactions まで。長期記憶は relationship と別に持つ |
