Aliasレイヤー

作品名やキャラクター名は議論の参照元にできますが、公開実装ではオリジナル alias を使います。これにより、public repository としてレビュー可能で拡張しやすい状態を保ちます。

Alias機能最初の根拠
sentinel_mvp実行時 role takeover と containment pressure。Role event、TTL、exit reason。
bridge_agentvirtual、real、liminal の world layer 間を移動する。World-layer transition log。
guide_agentルールを説明し、transition を提案する。Fallback 付き rule explanation。
operator_agent高信頼実行と rescue path を扱う。Human gate を意識した action log。
sentinel_swarm協調 monitoring と stale reporting を行う。Heartbeat と orphan threshold。

当初 docs-only PR から始め、現在は決定論的な runtime primitive、viewer affordance、motif docs、skill drift gate までを1つの reviewable slice にまとめています。

三つの世界

すべての transition は entry condition、exit condition、cost、replay evidence を持つ必要があります。

real

接地レイヤー

Visible map state、deterministic replay、public demo behavior、data contract を扱う層。

virtual

シミュレーションレイヤー

Rule game、generated scenario、role takeover、alternate run、benchmark を扱う層。

liminal

境界レイヤー

Hidden memory、transition state、夢的な解釈、uncertain identity を扱う境界層。

進捗報告運用

作業開始時にTODOを確認し、キリの良い単位で status と証拠を更新します。各作業セクションの終わりに「今回の進捗」「確認できたこと」「次の予定」を残します。

開始時

今回触る TODO を 進行中 にし、依存関係と stop 条件を確認する。

更新時

完了 / 保留 の理由と証拠を roadmap または issue に残す。

報告時

実装内容、変更ファイル、検証、未完了項目、次の推奨作業をまとめる。

やらないこと

創造性は残します。ただし、公開実装には明確な停止線を置きます。

停止線

  • 保護されたキャラクター名を code identifier にコピーしない。
  • 保護された引用を trigger として保存しない。
  • lookalike art、voice、music、official-looking asset を生成しない。
  • real-person impersonation を実装しない。
  • human review なしで public PR、deployment、secret change、costly cloud run を行わない。
