## 伝えたい芯

`urban-ecosystem` は、完成品を眺めるだけのデモではなく、触りながら「どこで失敗するか」と「どう直せるか」を覚えるための小さな実験場にしたい。

コードを書ける人だけの場所ではなく、README を読んで迷う、起動して詰まる、画面を見て意味が分からない、設定が怖い、課金境界が不安、という反応もそのまま価値にします。

公開してよかった一番大きな出来事は、実際に外から PR を作ってもらえたことです。完成度を競うより、誰かが GitHub を触る、README を読む、ローカルで動かす、壊す、直す、Issue や PR に慣れるための場として機能すれば、それでかなり成功です。

失敗が出たときに、原因を責めるのではなく、次の人が同じ場所で迷わないように何を直すかを残していくことで、プロジェクトそのものを学習できる場所にします。

## Discord 投稿案

`urban-ecosystem` は、渋谷の街を舞台にした AI エージェントの都市シミュレーションです。

ただ、今いちばんやりたいのは「すごいデモを見てください」というより、誰でも触って、好きな角度から壊したり迷ったりしながら、失敗の仕方と直し方を身につけられる場所にすることです。

たとえば、コードを書かなくても大丈夫です。

- README を読んで、最初にどこで迷ったか
- API キーなしの fallback viewer を見て、何が分かりにくいか
- Google Cloud や課金境界の説明で怖いところがあるか
- 街データとして何を入れると面白そうか

こういうコメントが、そのまま改善材料になります。

このリポジトリは、最初から Google Cloud や API キーを使わなくても触れるようにしています。まずは安全な fallback viewer から試せます。

将来的に Google Cloud 側を触りたい人が出てきたら、申請があった場合にだけ、こちらで範囲を切って提供する予定です。いきなり全員に gcloud 前提で開くのではなく、まずはローカルで試せる練習台として置いておきます。

あと、公開して本当によかったと思ったのは、実際に外から PR を作ってもらえたことです。自分にとっては、それだけで何年分かの価値がありました。

まだ公開して短い期間なのに、その間に GitHub の使い方、Issue / PR / review の流れ、公開 docs の整え方、安全境界の引き方まで一気に触ることになりました。かなり圧縮された学習だったと思います。

## 話す順番

1. 何を作っているか
   - 渋谷を舞台にした AI エージェント都市シミュレーション。
   - 住人が通勤、昼食、買い物、交流、帰宅をして、その一日を viewer で見られる。
2. 何を目指しているか
   - 完成品の鑑賞ではなく、触って学べる場所。
   - 失敗、迷い、分かりにくさを改善材料として扱う。
   - 自分の手を離れて、GitHub やローカル実行の練習台として育つ状態を目指す。
3. なぜ fallback viewer が大事か
   - 最初から Google Cloud や API キーを要求すると、参加の失敗が課金や秘密情報の不安に寄ってしまう。
   - API キーなしでレビューできる入口があると、画面、文言、操作、README の問題に集中できる。
   - gcloud は最初から全員に開くのではなく、希望者が出たら申請ベースで範囲を切って提供する。
4. 何を見て作ったかの裏話
   - 着想元は、公開 SNS で見た都市上の AI エージェント生態系っぽい投稿と動画。
   - 元動画の目的や作者の意図を完全に理解していたわけではない。
   - 見えた要素を、API キーなし・fallback viewer・データ契約つきの実験場に変換した。
5. 最初にお願いしたいこと
   - 大きな実装ではなく、まずは触って「ここで迷った」を教えてほしい。
   - API キー、token、`.env`、個人情報は貼らない。
   - 正式な判断や採否は GitHub issue / PR に残す。

## Source note

公開してよい範囲で言える根拠:

- 公開 SNS の投稿と添付動画を材料にした。
- 動画から読み取れた要素を整理し、文字起こし、仕様メモ、work order、repo 実装へ進めた。
- 元動画の作者の目的や意図を完全に理解していたわけではない。
- この repo は、見えた要素をそのまま複製するのではなく、API キーなし・fallback viewer・データ契約つきの実験場へ変換したもの。

言いすぎないほうがよいこと:

- 元動画の作者の意図や目的を理解していた、とは言わない。
- 元動画そのものを完全再現した、とは言わない。
- 大規模対応や都市開発応用を実装済み、とは言わない。
- gcloud を無条件に開放する、とは言わない。申請ベース・範囲限定・人間確認ありにする。

## 注意

- Discord は軽い案内と会話の入口にする。
- 決定、採否、完了条件は GitHub issue / PR / docs に残す。
- API キー、token、`.env`、Webhook URL、内部 URL、個人情報は投稿しない。
- 公開デモや repository visibility の扱いは、別途 human review と明示承認を通す。
